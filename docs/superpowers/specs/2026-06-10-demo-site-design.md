## Overview

Add a `demo/` Docusaurus site workspace to the `docusaurus-plugin-faqs` repo, following the same pattern as `docusaurus-plugin-plausible`. The demo site serves as the canonical documentation home for the plugin and demonstrates its capabilities live — the FAQ page at `/faqs` is itself generated by the plugin from YAML files in the workspace, making it a self-demonstrating site. The existing README is simplified to a pointer toward the demo site.

## Demo Site Content

### Landing page (`/`)

Hero section with plugin name ("FAQs for Docusaurus"), tagline ("Build a categorised FAQ page from YAML files"), and CTA buttons to Docs and the live FAQ page. Feature grid below:

| Feature | Description |
| ------- | ----------- |
| YAML-driven | Write questions and answers in plain YAML — no React, no MDX |
| Categories & ordering | Group questions under headings, control sort order explicitly |
| Direct links | Every question gets a stable permalink; visiting the URL auto-expands it |
| i18n-ready | Locale-aware fallback directory for translated FAQ files |

### Documentation (`/docs`)

Content migrated and expanded from the current README:

| File | Content |
| ---- | ------- |
| `intro.md` | What the plugin does, install command, minimal quick-start config (TS and JS variants) |
| `configuration.md` | Both options — `path` and `routeBasePath` — with types, defaults, and descriptions |
| `writing-faqs.md` | YAML format, all 4 fields (`question`, `answer`, `category`, `order`), skip/warn behaviour for invalid files |
| `direct-links.md` | How slugs are derived from question text, the hover permalink `#`, auto-expand and scroll on load, duplicate slug disambiguation |
| `internationalisation.md` | The `i18n/<locale>/docusaurus-plugin-faqs/` fallback directory, how to provide translated FAQ YAML files |

### Live FAQ page (`/faqs`)

Generated entirely by the plugin from YAML files in `demo/data/faqs/`. No custom React page — this IS the demonstration. Four categories of meta FAQs about the plugin itself, with answers that link back to the relevant docs pages where appropriate.

**Getting Started**

- How do I install the plugin?
- How do I add it to my Docusaurus config?
- Where do I put my FAQ files?

**Writing FAQs**

- What fields does a FAQ entry support?
- How do I group questions into categories?
- How do I control the order of entries within a category?
- What happens if I omit `category`?

**Features**

- How do direct links work?
- What Markdown is supported in answers?
- Does the plugin work with Docusaurus i18n?

**Troubleshooting**

- My FAQ file is being skipped — why?
- All my questions appear under "General" — what's wrong?
- The FAQ page is empty but my YAML files exist

Each answer is 2–4 sentences, self-contained, and links to the relevant docs page (e.g. "→ See [Configuration](/docs/configuration)") where a fuller explanation lives.

## Workflow Improvements

### `deploy-demo.yml` (new)

Adapted directly from `docusaurus-plugin-plausible`'s `deploy-demo.yml`:

- **Triggers:** push to `main`, `pull_request` (non-fork), `pull_request_target` with `request-deploy` label (forks), `workflow_dispatch`
- **Permissions:** `contents: write`, `deployments: write`, `pull-requests: write`
- **Concurrency:** cancel-in-progress per branch
- **Timeout:** 15 minutes
- **Steps:** checkout (with `fetch-depth: 0`) → corepack → node (current) → `yarn install --immutable` → `yarn build` (plugin) → `yarn workspace docusaurus-plugin-faqs-demo build` → Cloudflare Pages deploy
- **Post-deploy:**
  - On `main` push: commit comment with deployment URL
  - On PR: PR comment with preview URL
  - GitHub Step Summary with deployment table on every run
- **Fork safety:** separate `comment` job that posts a "needs `request-deploy` label" message when a fork PR is first opened

**Cloudflare Pages config:**

- Project name: `docusaurus-plugin-faqs`
- Build output: `demo/build`
- Production URL: `faqs.docusaurus.homotechsual.dev`
- Secrets required: `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_API_TOKEN`

### `publish.yml` (updated)

Add `packages: write` to top-level permissions. Add a second job `publish-gpr` (needs: `publish-npm`) that publishes to the GitHub Package Registry (`https://npm.pkg.github.com`) using `GITHUB_TOKEN`.

### `ci.yml` (updated)

Add `yarn test` step with `continue-on-error: true`. The current `test` script exits 1 as a placeholder; `continue-on-error: true` keeps CI green while making the failure visible as a reminder.

## Deployment Prerequisites (manual steps)

These are not code changes but must be done before the deploy workflow succeeds:

1. Create Cloudflare Pages project `docusaurus-plugin-faqs` (or confirm it already exists)
2. Configure custom domain `faqs.docusaurus.homotechsual.dev` in Cloudflare
3. Add GitHub secrets `CLOUDFLARE_ACCOUNT_ID` and `CLOUDFLARE_API_TOKEN` to the repo (same values used by other plugins)
